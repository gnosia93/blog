라비톤 5(M9g 인스턴스)의 **SME(행렬 연산 엔진)**와 대폭 커진 L3 캐시(192MB), 그리고 BF16 포맷의 시너지를 가장 극적으로 받아먹는 '작은 모델'들은 명확합니다.
GPU를 쓰기엔 아깝고, 구형 CPU로 돌리면 버벅이던 **1B(10억) ~ 8B(80억) 파라미터 사이의 소형 언어 모델(SLM)**과 임베딩/비전 모델들이 그라비톤 5 SME의 최대 수혜자입니다.


### 1. 1B ~ 3B 급: 초경량·실시간 에이전트용 SLM ###
AWS가 그라비톤 5를 출시하며 가장 강조한 영역이 `에이전트 AI(Agentic AI)` 이다. 복잡한 툴을 호출하고 다음 행동을 실시간으로 판단하는 에이전트 시스템에서는 1초 미만의 반응 속도(초저지연)가 생명입니다.
•	Llama 3.2 1B / 3B: 메타의 대표적인 경량 모델입니다. 그라비톤 5의 SME 기능을 이용해 이 모델을 BF16이나 INT8로 돌리면, CPU 내부에서 ‭$64 \times 64$‬‭‬ 행렬 타일로 완벽하게 쪼개져 처리됩니다. GPU로 데이터를 주고받는 오버헤드가 없기 때문에 글자 한 자가 출력되는 첫 번째 시간(TTFT)이 극도로 짧아집니다.
•	Gemma 2 2B: 구글의 경량 모델로, 가볍지만 지능이 높아 온디바이스나 CPU 기반 에이전트 환경에서 자주 쓰입니다.
•	Microsoft Phi-3.5 mini (3.8B): 추론 능력이 뛰어난 소형 모델로, 그라비톤 5 환경에서 다수의 사용자 요청을 동시에 처리하는 웹 서버 기반 AI 봇을 가성비 있게 구성할 때 최적입니다.
